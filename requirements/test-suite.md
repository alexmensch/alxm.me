## Feature: Comprehensive Test Suite

The suite uses the Node.js built-in test runner and `assert/strict`, matching existing conventions, with all tests consolidated into a single `tests/` directory.

## Out of scope

- **Template rendering tests**: Testing that Liquid/Nunjucks layouts produce specific HTML output. Content changes frequently and visual correctness cannot be verified through string matching.
- **CSS/SCSS output tests**: Stylelint already validates SCSS syntax and conventions. Automated testing of visual appearance is not feasible without visual regression tooling, which is disproportionate for a personal site.
- **Favicon generation script**: `scripts/generate-favicons.js` produces image files that would require visual comparison or pixel-level assertions. Not practical to unit test.
- **R2 sync script**: `_cloudflare/r2/sync.js` is a deployment utility that interacts with external Cloudflare APIs. Testing would require extensive mocking for little value.
- **Content correctness**: Individual blog posts, podcast episode metadata, and artwork entries are content, not application logic. They don't benefit from automated testing.
- **Cloudflare KV interactions**: The permalink checker's KV storage layer and the KV collections plugin are integration concerns that depend on external services. Only the local diff logic is tested.
- **End-to-end / integration tests**: Full Eleventy build output testing (building the site and inspecting the output) is not included in this scope. This could be a future enhancement but is a separate concern from unit testing application logic.
- **Visual regression testing**: Tools like Percy, Playwright screenshots, or BackstopJS for visual diff testing are not included. Could be considered as a separate future feature if visual regressions become a problem.
